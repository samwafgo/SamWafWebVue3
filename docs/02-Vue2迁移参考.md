# Vue2 → Vue3 迁移参考

> 老项目：`C:\huawei\goproject\SamWafWeb`（Vue 2.6 + TDesign Vue + Vuex + Vue Router 3）
> 新项目：`C:\huawei\goproject\SamWafWebVue3`（Vue 3.4 + TDesign Vue Next + Vite 5 + TS）

## 老项目目录结构（迁移对照源）

```
SamWafWeb/src/
├── apis/          # API 请求封装（按模块划分）
├── assets/        # 静态资源
├── bus/           # 事件总线（Vue3 中需替换：mitt 或 provide/inject）
├── components/    # 公共组件
├── config/        # 全局配置（主题色等）
├── constants/     # 常量定义
├── i18n/          # 国际化封装
├── layouts/       # 布局组件（侧边栏/头部/面包屑）
├── locales/       # 语言包（中/英）
├── pages/         # 页面（dashboard/login/user/waf/list/form/result/detail/nest-menu）
│   └── waf/       # WAF 核心业务页面（主迁移目标）
├── router/        # 路由（Vue Router 3 → 4）
├── service/       # 服务层
├── store/         # Vuex 3 → 建议改为 Pinia
├── style/         # 全局样式（less）
├── utils/         # 工具（加密/请求封装等）
├── main.jsx       # 入口
└── permission.js  # 路由守卫/权限控制
```

| vue-i18n ^8 | vue-i18n ^9+（待装） | `createI18n`，legacy: false |
| axios ^0.27 | axios（待装，可升新版） | 请求封装可大部分复用 |
| crypto-js | crypto-js（待装） | 接口加解密逻辑直接复用 |
| dompurify | dompurify（待装） | XSS 防护，必须保留 |

## 骨架未迁移项说明

骨架版 `request.ts` 相比老项目精简了以下逻辑，迁移对应功能时需补回：

1. **远控机器**：`localStorage.current_server` 存在时附加 `Remote-Waf-User-Id: {tenant_id}@{user_code}` 头（白名单接口除外：`/center/list`、`logout`、`public/login`）。
2. **403 页面**：HTTP 403 时整页渲染 Access denied。
3. **请求重试**：老项目有 retry=3 的退避重试。

老项目 layouts 中尚未迁移的功能（按需逐步补）：

- AI 助手聊天抽屉（t-sticky-tool + SSE 流式对话，`utils/eventSource.js`）
- 多标签页（tabRouter，Vuex + localStorage 持久化）
- top / mix 布局模式与 setting 设置面板（新版暂只做 side 布局）
- 头部组件：Notice 通知、Search 搜索、SystemMonitor 系统监控、在线文档链接

## 安全要求（与后端 CLAUDE.md 一致）

- 用户输入渲染前必须经过 DOMPurify 处理，防 XSS。
- 不在前端代码/日志中输出密码、Token、敏感请求体。
- 与后端通讯的加密逻辑（crypto-js）保持与 Vue2 版本一致，不得绕过。
